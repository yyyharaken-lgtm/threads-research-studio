# results/

リサーチ結果が自動保存される場所。

## フォルダ構成

```
results/
├── by-genre/             ← ジャンルモードの結果
│   ├── 美容/
│   │   └── 2026-05-04_0930.json
│   └── 副業/
│       └── 2026-05-04_1020.json
│
└── by-account/           ← アカウントモードの結果
    └── biyou._.m/
        └── 2026-05-04_1100.json
```

## ファイル形式

JSON形式。各ファイルは1回のリサーチ結果（10投稿前後）を含む。

ファイル名規則：`{日付}_{時刻}.json` 形式。
同じジャンル/アカウントを1日に複数回リサーチしても上書きされない。

## 注意

- このフォルダの中身はサーバーが自動生成する
- 不要になった結果は手動で削除してOK
- サイトUIには「履歴一覧」を出さない設計（必要ならフォルダから直接見る）
- 将来の分析ツールはここのファイルを読みに来る
