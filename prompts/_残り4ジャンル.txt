## ⚠️ 重要：ハイブリッド方式について

本リサーチは**Phase 1（キーワード探索）+ Phase 2（フォロー欄展開）**のハイブリッド方式で実施します。
Phase 1だけだと取りこぼしが多いので、必ず Phase 2 まで実行してください。

## 各ジャンルでの作業手順

各ジャンルについて、以下を必ず実行してください：

### Step A: 開始宣言
「【ジャンル N/4: <ジャンル名>】を開始します」と表示。

### Step B: 判定基準を読み込む
そのジャンルの判定基準ファイル（howto-judgment.md または uranai-judgment.md）を Read ツールで読む。

### Step B': 既取得URLリストの作成（重複防止）
**毎回必ず実行すること。** これがないと同じ投稿を何度も拾うことになる。

1. `results/by-genre/<ジャンル名>/` 配下の **全JSONファイル** を Glob または Read で取得
2. 各JSONファイルを開いて、`posts` 配列内の `post_url` をすべて収集
3. それらを「**既取得URLリスト**」としてメモリに保持

例：
- 「`results/by-genre/美容/` には3ファイルあって、合計28件のURL が既に取得済み」
- これらのURLは今回スキップ対象

該当ジャンルのフォルダがまだ存在しない場合（=初回）は、空のリストでOK。

### Step C: Phase 1 — キーワード探索（既取得URLは必ずスキップ）
1. https://www.threads.com を開く（既に開いていればそのまま）
2. リサーチ用のChromeプロファイルで動作していることを確認
3. 検索キーワードを順番に検索する
4. 各キーワードについて：
   - 上部の検索バーをクリック
   - キーワードを入力して検索
   - 「投稿」タブを選択
   - 上から順にスクロールして投稿を確認
   - 各投稿について：
     - **【最重要】投稿URLが「既取得URLリスト」に含まれていないか** ← Step B' のリスト
       - 含まれている場合 → 即スキップ、次の投稿へ（時間とトークンの節約）
       - 含まれていない場合 → 次のチェックへ
     - **いいね数が100以上か**
     - **投稿日が直近2ヶ月以内か**
     - 上記をすべて満たす場合のみ次のステップへ
   - 投稿の発信者プロフィールを確認し、判定基準ファイルの
     アカウント判定スコアを算出
     - 7点以上 → 投稿を採用候補に追加
     - 6点以下 → スキップ
   - 採用候補の投稿について、投稿判定スコアを算出
     - 2点以上 → 正式採用、結果リストに保存
     - 1点以下 → 除外
   - 最大20投稿までスクロール確認したら、そのキーワードでの探索を終了
   - 次のキーワードへ移る
5. 採用投稿が10件に達した時点で、Phase 1 を終了して結果保存へ
6. Phase 1 で 10件に達しなかった場合 → **Phase 2 に進む**

### Step C2: Phase 2 — フォロー欄から優良アカウント発掘（Phase 1で10件未満の場合のみ）

Phase 1 で見つけた優良アカウント（=採用された投稿の発信者）を1つ選び、
その**フォロー欄からさらに優良アカウントを発掘**します。

1. 起点アカウント選定
   - Phase 1 で採用された投稿の発信者の中から1人選ぶ
   - 複数いる場合はフォロワー数が多い方を優先

2. フォロー欄を開く
   - 起点アカウントのプロフィールページを開く
   - 「フォロー中」リンクをクリック
   - フォロー先一覧を表示

3. フォロー先からの候補絞り込み（最大10件）
   - 一覧をスクロールしながら、bioと名前から「ジャンル関連」と判断できるアカウントを抽出
   - 優先順位：
     - bioでジャンルが明らかなアカウント
     - フォロワー数 5,000以上
     - 外部リンク（note/lin.ee 等）がある

4. 各候補アカウントを審査
   - プロフィールページを開く
   - 判定基準ファイルでアカウント判定スコアを算出
   - 7点以上 → 過去投稿を最大15件スクロール確認
   - 6点以下 → 次の候補へ

5. 通過アカウントの投稿から採用候補を抽出
   - 各投稿について：
     - URLが既取得URLリストに含まれていないか
     - いいね数100以上、直近2ヶ月以内
     - 投稿判定スコア2点以上
   - 全て満たす → 採用

6. 採用投稿が合計10件（Phase 1 + Phase 2の合計）に達した時点で終了
7. フォロー先10件を確認しても10件に満たない場合は、その時点の件数で結果出力

### Step D: 取得項目
各投稿について以下を取得：
- 投稿者ユーザー名
- 投稿者プロフィールURL
- 投稿本文（全文）
- いいね数
- コメント数（取得可能なら、なければ null）
- 投稿日（YYYY-MM-DD 形式）
- 投稿URL
- ヒットしたキーワード

### Step E: 結果保存
**【重要】JSON形式で** 以下のパスに保存：

`results/by-genre/<ジャンル名>/<今日の日付>_<現在時刻>.json`

例：`results/by-genre/育児/2026-05-05_0300.json`

ファイル全体は以下の構造：

```json
{
  "mode": "genre",
  "genre": "<ジャンル名>",
  "keywords": ["<キーワード1>", "<キーワード2>"],
  "executed_at": "YYYY-MM-DD HH:MM",
  "post_count": 10,
  "posts": [
    {
      "rank": 1,
      "author_name": "example_account",
      "author_url": "https://www.threads.com/@example_account",
      "post_text": "投稿本文を全文ここに",
      "likes": 412,
      "comments": 38,
      "posted_at": "2026-05-02",
      "post_url": "https://www.threads.com/...",
      "matched_keyword": "<該当キーワード>",
      "account_score": 8,
      "post_score": 3
    }
  ]
}
```

JSON構文の注意：
- 文字コード UTF-8
- 改行は `\n` でエスケープ
- ダブルクォートは `\"` でエスケープ
- 数値は数値型、コメント数取得不可なら null
- 該当0件でも `posts: []` の空配列で必ずファイル作成

### Step F: 完了報告
「【ジャンル N/4: <ジャンル名>】完了。<件数>件保存しました。」と簡潔に報告。

### Step G: 次のジャンルへ
次のジャンルに進む。すべて終わったら最終レポート（後述）を出す。

## エラー時の対応

- 1つのジャンルで失敗した場合 → そのジャンルの結果を `posts: []` の空配列で保存し、`error: "<エラー内容>"` フィールドを追加。次のジャンルへ進む。
- ネットワークが切れた場合 → 5分待って再試行。ダメなら次のジャンルへ。
- どこかで止まった場合 → 必ず最後に「ここまで完了したジャンル」を報告すること。

## 注意事項

- メイン垢では絶対に動かさない（リサーチ用Chromeプロファイル使用）
- 投稿スクロール時、1秒以上の間隔を空ける
- 該当する投稿が0件でも、その旨を結果ファイルに記録
- スクリーンショットは取得しない（テキスト抽出のみ）
- 投稿本文は省略せず全文取得
- 一気に全部やろうとして混乱しないこと。1ジャンルずつ確実に。
