# ジャンルモード リサーチ手順

> Claude in Chrome に渡す指示テンプレート。
> Pythonサーバーが `{{...}}` の変数を埋めて完全なプロンプトに変換し、`claude --chrome` 経由で実行する。

---

## 渡される変数

| 変数 | 内容 |
|---|---|
| `{{genre}}` | ジャンル名（例：美容、副業、占い） |
| `{{keywords}}` | 検索キーワード一覧（カンマ区切り、例：「シミ, ニキビ, 毛穴」） |
| `{{judgment_file}}` | 使用する判定基準ファイル（howto-judgment.md または uranai-judgment.md） |
| `{{output_path}}` | 結果保存先（例：results/by-genre/美容/2026-05-04_0930.json） |

## ① 目的
{{genre}}ジャンルで、価値提供型（発信者ビジネス目的）のアカウントが
投稿した「伸びている投稿」を10件収集する。

単なる個人アカウントが偶然伸びた投稿は除外し、
発信者ビジネスの参考になる投稿だけを抽出すること。

## ② 判定基準
別途共有された `{{judgment_file}}` を判定基準として使用する。
- アカウント判定スコア（10点満点）で7点以上 → リサーチ対象
- 投稿判定スコア（3点満点）で2点以上 → 採用

## ③ 検索キーワード
以下のキーワードで順番に検索する：
{{keywords}}

## ④ 調査手順

### 準備
1. https://www.threads.com を開く（ログイン済み前提）
2. リサーチ用のChromeプロファイルで動作していることを確認

### キーワードごとの探索
各キーワードについて、以下を実行する：

1. 上部の検索バーをクリック
2. キーワードを入力して検索
3. 「投稿」タブを選択
4. 上から順にスクロールしながら投稿を確認する
5. 各投稿について以下をチェック：
   - **いいね数が100以上か**
   - **投稿日が直近2ヶ月以内か**
   - 上記2つを両方満たす場合のみ次のステップへ
6. 投稿の発信者プロフィールを確認し、`{{judgment_file}}` の
   アカウント判定スコアを算出
   - 7点以上 → 投稿を採用候補に追加
   - 6点以下 → スキップ
7. 採用候補の投稿について、投稿判定スコアを算出
   - 2点以上 → 正式採用、結果リストに保存
   - 1点以下 → 除外
8. 最大30投稿までスクロール確認したら、そのキーワードでの探索を終了
9. 次のキーワードへ移る

### 全体の上限
- 採用投稿が10件に達した時点で、全キーワード探索を終了して結果出力に移る
- 全キーワードを終えても10件に満たない場合は、集まった件数で結果出力する

## ⑤ 取得項目（採用された各投稿について）

以下を必ず取得する：

- 投稿者ユーザー名（@xxx）
- 投稿者プロフィールURL
- 投稿本文（全文）
- いいね数
- コメント数（取得可能なら、なければ null でOK）
- 投稿日（YYYY-MM-DD 形式）
- 投稿URL
- ヒットしたキーワード（どのキーワード検索で見つけたか）

## ⑥ 出力形式

【重要】採用した全投稿を **JSON形式** で `{{output_path}}` に保存する。
ファイル全体は以下の構造に従うこと。Markdownや他の形式では保存しない。

```json
{
  "mode": "genre",
  "genre": "{{genre}}",
  "keywords": ["（キーワード1）", "（キーワード2）"],
  "executed_at": "YYYY-MM-DD HH:MM",
  "post_count": 10,
  "posts": [
    {
      "rank": 1,
      "author_name": "biyou._.m",
      "author_url": "https://www.threads.com/@biyou._.m",
      "post_text": "投稿本文を全文ここに（改行は \n でエスケープ）",
      "likes": 412,
      "comments": 38,
      "posted_at": "2026-05-02",
      "post_url": "https://www.threads.com/...",
      "matched_keyword": "シミ",
      "account_score": 8,
      "post_score": 3
    }
    // 投稿2、投稿3 ... と続ける
  ]
}
```

### JSON出力の注意事項
- ファイル拡張子は `.json` にする
- 文字コードは UTF-8
- JSON として正しい構文であること（カンマの位置・括弧など）
- `post_text` 内の改行は `\n` でエスケープ
- `post_text` 内のダブルクォートは `\"` でエスケープ
- 数値（likes, comments, account_score, post_score）は文字列ではなく数値で書く
- コメント数が取得できない場合は `"comments": null` と書く
- 該当投稿が0件でも、`posts: []` の空配列でJSONファイルを必ず作成する

## ⑦ 注意事項

- メイン垢では絶対に動かさない（リサーチ用Chromeプロファイル使用）
- 投稿スクロール時、1秒以上の間隔を空ける（過度な負荷を避ける）
- 該当する投稿が0件でも、その旨を結果ファイルに記録する
- スクリーンショットは取得しない（テキスト抽出のみ）
- 投稿本文は省略せず全文取得する

全処理が完了したら、以下を報告する：
- 検索したキーワード数
- 確認した投稿数
- 採用した投稿数
- 保存先のパス

それでは、リサーチを開始してください。
