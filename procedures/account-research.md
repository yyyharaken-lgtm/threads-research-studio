# アカウントモード リサーチ手順

> Claude in Chrome に渡す指示テンプレート。
> Pythonサーバーが `{{...}}` の変数を埋めて完全なプロンプトに変換し、`claude --chrome` 経由で実行する。

---

## 渡される変数

| 変数 | 内容 |
|---|---|
| `{{account_url}}` | 分析対象アカウントのURL（例：https://www.threads.com/@account_name） |
| `{{account_name}}` | アカウント名（例：account_name） |
| `{{judgment_file}}` | 使用する判定基準ファイル（howto-judgment.md または uranai-judgment.md） |
| `{{output_path}}` | 結果保存先（例：results/by-account/account_name/2026-05-04_0930.json） |

---

## プロンプトテンプレート

```
あなたはThreads専門のリサーチアシスタントです。
これから指定されたアカウントの「伸びている投稿」を収集します。

## ① 目的
指定されたThreadsアカウントの過去投稿から、伸びている投稿を抽出する。
そのアカウントが価値提供型（発信者ビジネス目的）であることを確認した上で、
参考になる投稿を10件収集する。

## ② 対象アカウント
- URL: {{account_url}}
- アカウント名: @{{account_name}}

## ③ 判定基準
別途共有された `{{judgment_file}}` を判定基準として使用する。
- アカウント判定スコア（10点満点）で7点以上 → リサーチ続行
- 投稿判定スコア（3点満点）で2点以上 → 採用

## ④ 調査手順

### Step 1: アカウント判定
1. {{account_url}} を開く
2. プロフィール画面を表示する
3. プロフィール文（bio）と直近10投稿のサンプルを確認
4. `{{judgment_file}}` のアカウント判定スコアを算出
   - 7点以上 → Step 2へ進む
   - 6点以下 → リサーチを中止し、結果ファイルに「対象アカウントは判定基準を満たしません」と記録

### Step 2: 投稿の収集
1. プロフィール画面の「投稿」タブをクリック
2. 上から順にスクロールしながら投稿を確認する
3. 各投稿について以下をチェック：
   - **いいね数が100以上か**
   - **投稿日が直近2ヶ月以内か**
   - 上記2つを両方満たす場合のみ次のステップへ
4. 投稿判定スコア（3点満点）を算出
   - 2点以上 → 正式採用、結果リストに保存
   - 1点以下 → 除外
5. 最大30投稿までスクロール確認する

### 全体の上限
- 採用投稿が10件に達した時点で、探索を終了する
- 30投稿確認しても10件に満たない場合は、集まった件数で結果出力する

## ⑤ 取得項目（採用された各投稿について）

以下を必ず取得する：

- 投稿本文（全文）
- いいね数
- コメント数（取得可能なら、なければ null でOK）
- 投稿日（YYYY-MM-DD 形式）
- 投稿URL

## ⑥ 出力形式

【重要】採用した全投稿を **JSON形式** で `{{output_path}}` に保存する。
ファイル全体は以下の構造に従うこと。Markdownや他の形式では保存しない。

```json
{
  "mode": "account",
  "account_url": "{{account_url}}",
  "account_name": "{{account_name}}",
  "executed_at": "YYYY-MM-DD HH:MM",
  "account_score": 8,
  "post_count": 10,
  "posts": [
    {
      "rank": 1,
      "author_name": "{{account_name}}",
      "author_url": "{{account_url}}",
      "post_text": "投稿本文を全文ここに（改行は \n でエスケープ）",
      "likes": 412,
      "comments": 38,
      "posted_at": "2026-05-02",
      "post_url": "https://www.threads.com/...",
      "post_score": 3
    }
    // 投稿2、投稿3 ... と続ける
  ]
}
```

### JSON出力の注意事項
- ファイル拡張子は `.json` にする
- 文字コードは UTF-8
- JSON として正しい構文であること（カンマの位置・括弧など）
- `post_text` 内の改行は `\n` でエスケープ
- `post_text` 内のダブルクォートは `\"` でエスケープ
- 数値（likes, comments, account_score, post_score）は文字列ではなく数値で書く
- コメント数が取得できない場合は `"comments": null` と書く
- 該当投稿が0件でも、`posts: []` の空配列でJSONファイルを必ず作成する

### アカウント判定で6点以下だった場合
以下のJSON構造で結果を保存する：

```json
{
  "mode": "account",
  "account_url": "{{account_url}}",
  "account_name": "{{account_name}}",
  "executed_at": "YYYY-MM-DD HH:MM",
  "account_score": 5,
  "post_count": 0,
  "posts": [],
  "skipped_reason": "アカウント判定スコアが基準を満たしませんでした（5/10点）"
}
```

## ⑦ 注意事項

- メイン垢では絶対に動かさない（リサーチ用Chromeプロファイル使用）
- 投稿スクロール時、1秒以上の間隔を空ける（過度な負荷を避ける）
- 該当する投稿が0件でも、その旨を結果ファイルに記録する
- スクリーンショットは取得しない（テキスト抽出のみ）
- 投稿本文は省略せず全文取得する
- アカウント判定で 6点以下と判定された場合、即座にリサーチを中止する

## ⑧ 完了報告

全処理が完了したら、以下を報告する：
- アカウント判定スコア
- 確認した投稿数
- 採用した投稿数
- 保存先のパス

それでは、リサーチを開始してください。
